# 랜딩페이지 알림톡 및 광고 전환 추적 연동

## 포트폴리오 개요

랜딩페이지 신청 폼 제출 이후 알림톡 발송과 광고 전환 이벤트가 실행되도록 연동하는 구조를 보여주는 데모입니다. 고객 정보 저장 완료 시점을 기준으로 알림톡 API, Meta Pixel, GA4, Google Ads 전환 추적을 연결하는 상황을 가정했습니다.

이 데모는 실제 API 키 없이 mock 함수로 동작하며, 위시켓 포트폴리오 제출을 위한 유사 구조 구현 예시입니다.

## 작업 범위

- 신청 폼 UI 구현
- 알림톡 발송 API 호출 구조 설계
- 전환 이벤트 실행 시점 정의
- Meta Pixel, GA4, Google Ads 이벤트 mock 처리
- 성공/실패 로그 출력

## 주요 기능

- 신청자 이름/연락처/문의 유형 입력
- 제출 완료 시 알림톡 mock 발송
- 광고 전환 이벤트 mock 실행
- 처리 로그 화면 표시
- 중복 전환 방지를 고려한 완료 시점 분리

## 기술적 주안점

광고 추적은 단순 태그 삽입보다 정확한 전환 시점 정의가 중요합니다. 이 데모는 폼 제출 완료 이후 알림톡 발송과 전환 이벤트가 순차적으로 실행되는 구조를 기준으로 했습니다.

실서비스에서는 Meta Events Manager, Google Tag Assistant, GA4 DebugView 등을 통해 이벤트 수집 여부와 중복 전환 여부를 검증하는 흐름을 적용할 수 있습니다.

## 실서비스 확장 방향

- 카카오 알림톡 API 또는 발송 대행사 API 연동
- CRM 저장 완료 이벤트와 알림톡 트리거 연결
- Meta Pixel Lead 이벤트 적용
- GA4 `generate_lead` 이벤트 적용
- Google Ads 전환 스니펫 적용
- 실패 로그 및 재발송 관리
